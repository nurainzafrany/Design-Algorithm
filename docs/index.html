About The Project

Disaster Response

During large-scale floods in Southeast Asia, rural towns become isolated when infrastructure is damaged. Our system ensures they receive emergency supplies.

Drone Optimization

We solve the critical problem of efficiently using limited drones to deliver supplies to all affected towns while respecting operational constraints.

Graph Algorithm

Using graph theory and recursive backtracking, we model towns as nodes and flight paths as edges to find optimal delivery routes.

Problem Definition

Southeast Asia has also seen more intense floods in recent years as a result of climate changes and other erratic weather conditions. Such floods can be very destructive to infrastructure by washing away roads, destroying bridges, and leaving whole towns isolated. In these cases of emergency, responding time of food, water, and medicines are of great importance. But when physical access is restricted, the traditional delivery method proves useless. In this situation drones provide an efficient and mobile approach.

The issue emerges on how to utilize a few numbers of drones to supply to all the affected towns effectively. An individual drone has a limit to the amount of help that it carries and limited ranges of flight using batteries and a particular town needs a given number of assistance. In addition, the drones will need to fly back to the base station once they have delivered the goods hence route optimization is critical to saving energy and time.

The task is to create the algorithm that designs the most economical set of the deliveries paths that enables drones with the possibility to pass through the towns, deliver necessary resources, and guard to base without overloading or infringing their limit. The algorithm also has to make sure that as many towns as possible are reached and the total travel distance minimized.

Graph Representation

To fix this, we represent the region of interest as a graph:

  • The nodes are presented as towns and the base, which are coordinated and need supplies.
  • Edges are lines encoding a direct flight distance, weighted by the Euclidean distances.
  • Discovering the most appropriate delivery path is practically a variant of the Traveling Salesman Problem (TSP) with extra two restrictions, namely capacity and range, thus it is a constrained combinatorial optimization problem.

System Output Requirements

Necessary output of the system is:

  • The towns of visit per route
  • Total miles and processing time per run.
  • Town-wise level of supply achievement
  • The measurements of efficiency, e.g., the consumption of energy

Theoretically and practically, this is a serious problem: it presents an implementation of the reality of actual logistics coupled with algorithmic solution-solving, exposing the importance of algorithm formulation in systems designated as life-critical.

Project Objectives

The primary aims of this project revolve around the designing and analysis of an appropriate algorithm in optimizing emergency drone delivery on flood-stricken areas. The following are objectives agreed upon on this study:

Graph Representation

To represent the disaster area in the form of a graph in which every town and the base station will be a node and the available flight routes will be measured using geographic distances as weighted edges.

Algorithm Analysis

To discuss the different paradigms of algorithms: Sorting, Divide and Conquer, Dynamic Programming, Greedy and Graph algorithms to decide which of them is the most appropriate to solve a drone route optimization problem.

Implementation

To create an effective solution in Java, it is important to apply the chosen algorithm that would be able to calculate the optimal delivery routes with a realistic set of constraints: drones have limited reach and cargo weight.

Performance Evaluation

To justify that the proposed algorithm is correct and study its performance, including its time complexity in the best, average and worst-case scenario using the asymptotic notation.

The goals are meant to guarantee that the eventual algorithmic solution is both implementable and sound in theory thereby facilitating efficient emergency response planning that is scalable.

Problem Specification

In response to a widespread flood disaster in Southeast Asia, many rural towns have become inaccessible due to destroyed infrastructure. The use of drones has been proposed as a modern and efficient solution to deliver emergency supplies. However, drone deployment is limited by two major operational constraints: limited flight range and supply load capacity. Thus, a well-defined optimization model is necessary to determine the most efficient delivery route that ensures the highest number of towns receive aid within the drone's constraints. This problem requires a combinatorial solution that integrates spatial modeling, route optimization, and constraint satisfaction within a computational framework.

4.1 Input Specifications

The input to the model consists of a set of towns and the specification of a drone. Each town is described using the following data types:

Town Data Structure

  • String name: A unique identifier for each town.
  • int x, y: Integer values representing 2D Cartesian coordinates.
  • int supplyNeeded: The required number of emergency kits.

Drone Specification

  • int maxRange: Maximum one-charge round-trip range.
  • int maxCapacity: Maximum number of emergency kits the drone can carry.

All towns are stored in a List<Town>, and distances between towns are precomputed and stored in a double[][] distanceMatrix, based on Euclidean geometry.

4.2 Constraints

To ensure feasible and safe delivery missions, the following constraints are enforced in the model:

Capacity Constraint

The cumulative amount of supplies delivered during a trip must not exceed the drone's maximum load (maxCapacity).

Range Constraint

The total distance of a round-trip (from base to towns and back) must not exceed the drone's battery range (maxRange).

Visit Constraint

Each town may only be visited once in a single route.

Return Constraint

The drone must always return to the base after completing the delivery route.

4.3 Objective Function

The primary objective is to minimize the total distance of a valid round-trip delivery route that starts and ends at the base while delivering supplies to the maximum number of towns within the drone's operational limits. This is mathematically equivalent to minimizing the sum of the Euclidean distances between each consecutive town in the route, including the return to base.

In the algorithm, this is achieved through recursive backtracking: the algorithm explores each valid route and calculates its total distance, keeping track of the shortest path that satisfies all constraints using a global variable minDistance. Optimization occurs by pruning infeasible branches and updating the best route whenever a better solution is found.

Mathematical Formulation

Minimize: Σ di,j for all (i,j) in route

Subject to:

  • Σ supplyi ≤ maxCapacity
  • Σ di,j ≤ maxRange
  • Route starts and ends at base

Algorithm Variables

  • minDistance: Tracks shortest valid route
  • currentDistance: Current path distance
  • currentLoad: Current supplies carried
  • visited: Tracks visited towns

Algorithm Paradigm Analysis

When designing an efficient drone delivery system to deliver emergency supplies, we took into consideration a number of algorithms including Sorting, Divide and Conquer (DAC), Dynamic Programming (DP), Greedy, and Graph Algorithms. A review of their appropriateness in the solving of our problem, as well as their strengths and weaknesses is below.

Solution Paradigm Strength Weakness
Sorting
  • Is useful in preprocessing, e.g. sorting the towns based on urgency, distance, or supply requirements.
  • Fast algorithm, e.g. Merge Sort and Quick Sort have a fast algorithmic time of O(n log n).
  • May be used in decision making with regard to delivery order.
  • Does not do route planning and does not graph, it is based delivery optimization.
  • Does not consider limiting properties, like drone battery or range.
  • Complex spatial relations among the towns cannot be modeled.
Divide and Conquer (DAC)
  • Divides jam-packed issues into smaller and solvable sub-regions.
  • Enables paralleling of drone flight paths in various areas.
  • Makes it possible to deal with large geographical areas.
  • Sub-solution merging can generate non-optima global routes.
  • Merging becomes complicated because of overlapping constraints such as battery life, range.
  • Needs vigilance in zone allocation so as to be efficient.
Dynamic Programming (DP)
  • Makes the best solutions in case of overlapping subproblems.
  • Is able to model tract battery life and battery range and deliveries.
  • Applicable in the solution of complex problems such as word Traveling Salesman Problem (TSP).
  • Large graphs like exponentials have high time and space complexity.
  • Hard to scale to real time or large-scale emergencies.
  • Difficult to apply as opposed to greedy or graph methods.
Greedy
  • Easy and quick, appropriate during emergency cases.
  • Derives satisfactorily good approximation solutions in time.
  • Competent in situations where speed of reaction is an important concern.
  • Nothing ensures global optimality.
  • Able to deliver poor local decisions, which impact the entire route.
  • Can be inefficient with complex constraints.
Graph Algorithm
  • Towns and paths (nodes and edges) are naturally modeled.
  • Such well-known algorithms like Dijkstra, A*, MST, TSP aid in route optimization.
  • Makes it easy to visualize, and periodically update things as conditions vary.
  • The solution of some problems(e.g., TSP) is NP-hard, i.e. there exists no feasible solution to the problem on a large graph.
  • Have to do with scalability with heuristic or approximate solutions.
  • The more the towns and redistribution routes, the higher the complexity.

Our Selected Approach

After careful analysis of these paradigms, we determined that a Graph Algorithm approach with recursive backtracking provides the best balance between optimality and computational feasibility for our specific problem constraints. This approach naturally models the geographic relationships between towns while respecting the drone's operational limitations.

Algorithm Design

Recursive Backtracking for TSP with Constraints

Our solution implements a recursive backtracking approach tailored to the Traveling Salesman Problem (TSP) with additional constraints for the drone's battery range and supply capacity.

Algorithm Flowchart

Figure: Flowchart of the recursive backtracking algorithm for drone delivery optimization

Algorithm Components

Graph Representation

Towns are modeled as nodes with coordinates (x,y) and supply requirements. Edges represent direct flight paths weighted by Euclidean distance between towns.

Recursive Backtracking

Systematically explores all possible routes while backtracking when constraints are violated, ensuring no valid path is missed.

Constraint Pruning

Immediately discards paths that exceed drone range or capacity, significantly reducing the search space.

Optimization Techniques

  • Early Termination: If a complete route is found that delivers to all towns within constraints, the search can terminate immediately.
  • Path Length Tracking: Maintains the shortest valid path found so far, allowing pruning of longer paths during exploration.
  • Supply Prioritization: Towns can be sorted by urgency or supply requirements to guide the search toward critical deliveries first.
  • Memoization: Caches previously computed subproblems to avoid redundant calculations in overlapping routes.

Real-World Adaptations

To handle practical emergency scenarios, the algorithm includes several adaptations:

  • Partial Deliveries: When full coverage isn't possible, prioritizes routes that serve the most critical towns.
  • Multiple Drones: Can be extended to coordinate multiple drones by partitioning towns among available drones.
  • Dynamic Updates: The graph can be updated in real-time as new information about town needs or drone availability arrives.
  • Terrain Factors: Edge weights can incorporate terrain difficulty or weather conditions beyond simple distance.

Algorithm Analysis

Time Complexity:

  • Worst-case: O(n!) - When no pruning occurs (visits all permutations)
  • Best-case: O(n) - When constraints limit to a single valid path
  • Average case: Depends on pruning effectiveness

Space Complexity: O(n) for recursion stack and storage of visited towns

Conclusion

In this project, we have addressed a real time problem which is how to optimize delivery of emergency supplies using drones in the flood affected areas in the event where the conventional transportation routes are not feasible. By representing the problem as a restricted form of the Traveling Salesman Problem (TSP), we maximized the use of system search algorithms that rely on graphs with several steps of recursive backtracking to find the delivery path that would be most efficient.

By design and analysis, we managed to design an operational Java implementation that unites drone constraints like the reach and ability to carry cargo, so that every solution violates practical operational constraints. The algorithm was successfully tested with different possible scenarios to ensure correctness or robustness of the algorithm to find optimal (or feasible) delivery routes, and to cover the failure cases through graceful error handling when the limits were exceeded.

Our solution algorithm was not only an algorithmically correct solution but it could be also practically adjusted and applied to dynamic and high stakes activities like post-disaster relief. Graph algorithms gave clarity and flexibility as well as the capability to either expand or adapt to new information.

Finally, the case of this project indicates the efficacy of algorithmic thinking to address life-altering logistical problems. It also portrays that we can utilize and implement the central concepts of the course (complexity analysis, data design structures, and problem modeling) to provide a functional system that could sustain decision-making in real-life emergency situations.
